# Project Context

## Project Name

Project Atlas (Working Name)

---

## Project Goal

Build a platform that becomes the knowledge and memory layer for software systems.

The platform should:

* Understand software projects
* Collect information from multiple sources
* Create relationships between information
* Build a searchable knowledge base
* Preserve project knowledge over time

The system should help developers understand a project faster.

---

# Current Scope

We are currently working only on the foundation.

Focus areas:

1. Repository Understanding
2. Documentation Collection
3. Knowledge Storage
4. Project Context Generation

Do not design or implement:

* Agent Swarms
* Autonomous Development
* Auto Remediation
* Self-Healing Systems
* Multi-Agent Workflows
* Production Automation

Those may be future phases.

---

# Development Approach

The human team is responsible for:

* Product decisions
* Architecture decisions
* Scope definition
* Prioritization

Codex is responsible for:

* Executing assigned tasks
* Generating code
* Generating documentation
* Refactoring code
* Creating tests
* Answering implementation questions

Do not make product decisions.

If information is missing, ask questions rather than making assumptions.

---

# Current Phase

Phase 1: Knowledge Foundation

Objective:

Given a software project, the platform should be able to understand and document:

* Repository structure
* Services
* Components
* Dependencies
* Infrastructure resources
* Deployment artifacts

Expected outcome:

A user can connect a project and receive a generated understanding of how the project works.

---

# Immediate Task Flow

Before implementing anything:

1. Understand the current task.
2. Ask questions if requirements are unclear.
3. Implement only the requested scope.
4. Avoid adding future features.
5. Keep solutions simple.
6. Prefer maintainability over complexity.

---

# Rule

Build only what is required for the current phase.

Do not optimize for future phases unless explicitly instructed.
